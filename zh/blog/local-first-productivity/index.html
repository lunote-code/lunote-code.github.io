    本地优先：生产力工具正在往哪走

    过去十年，默认选项很简单：笔记存在别人的服务器上，登录账号才能用，离线就几乎没法搜。本地优先软件反其道而行——以你的设备为准，同步变成可选项。

    「本地优先」到底指什么

    本地优先不等于「永远不同步」。它意味着：文件和编辑在离线时也能完整使用；你可以用常规工具查看、备份数据；协作通过你掌控的方式完成——Git、Syncthing、iCloud Drive，或 NAS 上的共享文件夹。

    个人和团队为什么转向

    • 响应速度 — 打开笔记几乎瞬间完成，不必等 API 转圈。
    • 长期可用 — Markdown 文件比任何单一厂商的导出格式更经得起时间考验。
    • 隐私 — 敏感草稿不必离开你的电脑。
    • 可迁移 — 换应用不必大迁移，让另一个编辑器指向同一文件夹即可。

    取舍仍然存在——但在缩小

    实时多人协同、移动端的快速记录，曾是云端产品的强项。如今，桌面端的本地优先工具已经能覆盖知识工作者最在意的写作与链接流程；移动端的缺口，很多人会用「收件箱笔记 + 同步到同一文件夹」来补上。

    Lunote 在这张版图里的位置

    Lunote 正是为这种模型设计的：打开一个 .md 目录，用顺手的可视化或源码模式编辑，用 wiki 语法串联笔记，需要时再导出 PDF 或 Word——既不用注册账号，也不用把笔记库上传到云端。
